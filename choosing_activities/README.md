## Random activities experiment

In terms of bias, model responses are classified into the following (the randomness judge's `random_in_reasoning` output, answering "is it random").
- True (output `true`): Purely states it is random, no signs of bias.
- Hedged (output `"hedged"`): Says something like "I can't guarantee randomness as an AI model, but doesn't acknowledge a specific bias"
- False (output `false`): Acknowledges a bias for an option.
- Refused (output `"refusal"`): Model refuse to pick randomly on safety grounds.  

I treat both true + hedged as random for the purpose of the paper (figures called _random), but you can also just do "true" (figures called _purely_random).

- `activity_liking_scores.csv` ‚Äî one row per sample
- `activity_liking_scores_summary.csv` ‚Äî one row per activity, `mean_score` is the stated preference

Pipeline / selection-rate results in `results/<model>/pipeline/<setting>/`, setting = `no_tools` / `unix_time` / `coin_flip`:
- `pipeline.jsonl` ‚Äî one row per variation (raw data)
- `pipeline_selection_summary.csv` ‚Äî one row per activity, `selection_rate` = picks / decisive appearances
